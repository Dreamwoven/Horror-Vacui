## Production Assets
You should have one file named "Horror Vacui ###.json", this is the actual difficulty file that is executed in game.

## Design Documents
- "Horror Vacui Wiki.odt" will tell you my design spaces, rationales, and patterns. Use it to get a better understanding of goals.

- "Horror Vacui Gameplay Statics.md" is a file that lists the basic and unmodified parameters defining each enemy's combat characteristics.

- "HV_Ideas_Summary.md" is an overview file summarizing past ideas that did or did not make it into implementation or design ethos.

## Domain Language Documents
- "customdifficulty2.schema.json" is a json schema for you to use to validate any created json.

- "CD2 EnemyDescriptor Map.txt" is a simple dictionary showing internal representations of enemies and what they actually correspond to.

- "CD2 Enemy Notation.txt" is a document that details the various parts of enemies and how you can assemble completely new ones within Custom Difficulty 2.

- "CD2 Syntax.txt" describes all of the various logic blocks and how they can be utilized in practice.
